Low-level stimulation

Ll – Send commands

The header smpt_ll_client.h needs to be included to use the low level functions.
Each command is send using the smpt_send_ll_*() functions. Depending on the command, a struct or the packet number is used as a parameter. To check if a struct is valid, the functions in smpt_ll_packet_validity.h (smpt_is_valid_ll_*()) can be used.

  • smpt_send_ll_init():
    • is the first command that needs to be send to use the low level.
    • initializes the stimulation.
    • requires that no other stimulation level is initialized.
  • smpt_send_ll_channel_config():
    • can be used to execute single pulses after the low level has been initialized.
    • notice also Stimulation information
  • smpt_send_ll_stop():
    • Ends the low-level stimulation mode.

Ll – Receive responses

Every command that has been sent generates a response (ack). This response contains the same packet number as the sent command and a result variable, which provides execution and error information.

  • smpt_new_packet_received():
    • can be used to check if a new packet has been received.
    • triggers the read of the serial device.
  • smpt_last_ack():
    • returns general information about the response.

Some commands have additional parameters. Use the following functions to get them:

  • smpt_get_ll_init_ack()
    • Get the parameters of Smpt_ll_init_ack (response to ll_init)
  • smpt_get_ll_channel_config_ack():
    • Get the parameters of Smpt_ll_channel_config_ack (response to ll_channel_config_ack)

Ll – Buffering and Timing

  • The device can buffer up to 10 commands in its internal buffer. This is useful for the execution of stimulation with a high frequency. After a command has been processed, the internal buffer can buffer another command.
  • The processing time for the execution of a ll_channel_config from the internal buffer depends on the number of points in the pulse, but is less than 1ms.
