General

The library version can be obtained using smpt_library_version().

General – Handle the serial device

The connection to the stimulator is created using a virtual serial port. For each device a Smpt_device struct needs to be created and passed to the API functions for internal processing. To open and close this port the following functions can be used:

  • smpt_open_serial_port():
    • opens the serial port.
    • This is required before any command can be send.
  • smpt_close_serial_port():
    • closes the serial port.
  • smpt_check_serial_port():
    • can be used to check if a serial port is available (Windows only).

General – Send commands

  • smpt_send_get_version_main():
    • requests the version of the firmware and the smpt-library.
  • smpt_send_get_stim_status():
    • requests stimulation informations (e.g. HV-Level, init status)
  • smpt_send_get_device_id():
    • requests the device ID.
  • smpt_send_get_battery_status():
    • requests the status of the rechargeable battery.
  • smpt_send_reset():
    • resets the device.

General – Receive responses

Every command that has been sent generates a response (ack). This response contains the same packet number as the sent command and a result variable, which provides execution and error information.

  • smpt_new_packet_received():
    • can be used to check if a new packet has been received.
    • triggers the read of the serial device. On each call, if available one packet is read.
  • smpt_last_ack():
    • returns general information about the response that has been read with the last call of smpt_new_packet_received().

Some commands have additional parameters. Use the following functions to get them:

  • smpt_get_get_version_main_ack()
    • Get the parameters of Smpt_get_version_ack (response to get_version_main)
  • smpt_get_get_stim_status_ack():
    • Get the parameters of Smpt_get_stim_status_ack (response to get_stim_status)
  • smpt_get_get_device_id_ack():
    • Get the parameters of Smpt_get_device_id_ack (response to get_device_id)
  • smpt_get_get_battery_status_ack()
    • Get the parameters of Smpt_get_battery_status_ack (response to get_battery_status)

General – Error handling

To get any information about possible errors during processing in the library, use smpt_init_error_callback() to set the callback function that is called by the library.

Example: 

void error_callback(const char *message)
{
    // This function is called if the library has errors.
    // Print the "message" to log them
}

{
    ...
    smpt_init_error_callback(&error_callback);
}
