Mid-level stimulation

Ml – Send commands

The header smpt_ml_client.h needs to be included to use the mid level functions.
Each command is send using the smpt_send_ml_*() functions. Depending on the command, a struct or the packet number is used as a parameter. To check if a struct is valid, the functions in smpt_ml_packet_validity.h (smpt_is_valid_ml_*()) can be used.

  • smpt_send_ml_init():
    • is the first command that needs to be send to use the mid level.
    • initializes the stimulation.
    • requires that no other stimulation level is initialized.
    • enables the HV (high-voltage, required for the stimulation).
  • smpt_send_ml_update():
    • starts the stimulation or updates the parameters after the mid level has been initialized.
    • notice also the Stimulation information
  • smpt_send_ml_get_current_data():
    • is the live signal that needs to be send to the device periodically.
    • If the command is not send to the device periodically (Timeout: 2 s), the stimulation pauses. To continue the stimulation another ml_update command needs to be send.
  • smpt_send_ml_stop():
    • stops the stimulation.
    • disables the HV (high-voltage).

Ml – Receive responses

Every command that has been sent generates a response (ack). This response contains the same packet number as the sent command and a result variable, which provides execution and error information.

  • smpt_new_packet_received():
    • can be used to check if a new packet has been received.
    • triggers the read of the serial device.
  • smpt_last_ack():
    • returns general information about the response.

Some commands have additional parameters. Use the following functions to get them:

  • smpt_get_ml_get_current_data_ack():
    • returns the parameters of Smpt_ml_get_current_data_ack (response to ml_get_current_data).
